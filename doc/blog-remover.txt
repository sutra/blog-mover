Blog Remover（Blog 搬家）

* 数据流向
源 BSP（BlogReader） -> Blog Remover -> 目标 BSP（BlogWriter）

* 搬家过程分为三步
1、BlogReader 从源 BSP 读取数据；
2、选择编辑需要导入目标 BSP 的 WebLog；
3、BlogWriter 将第二步选择的 WebLogs 写入目标 BSP。

* 程序部署及运行环境
服务器端： JDK 1.5, Servlet 2.4/JSP 2.0（e.g. jakarta-tomcat 5.5.17 http://tomcat.apache.org）
客户端： xhtml 1.1, Javascript, CSS 2.0

* 类图
见：ClassDiagram1.png

BlogReader：需要针对不同的 BSP 实现不同的 BlogReader，比如目前已经实现的 AnonymousSpaceReader
是采用 HTTP Get 的方式抓去数据，然后分析出其中的 WebLog 信息；SohuBlogReader 则是用过 Blogger 管
理自己的 WebLog 的界面来抓取数据。
这种实现方式的缺点是，抓去数据时需要针对不同的 BSP 解析 HTML 提取出其中的有用信息，有时由于部分 BSP 的
页面没有提供某个信息而需要抓取较多的HTML页面进行分析，比如 Sohu 的 Blogger 的发布 WebLog 的管理界面
就没有办法获取到 WebLog 的发布时间（注意时间不是日期），需要从 WebLog 的查看页面才能获取到发布时间。

BlogWriter：将 WebLog 写入目标 BSP，可以使用任何可能的方式实现，比如使用JDBC直接写入数据，
或者调用 Web Service 等接口。

* 如何从源 BSP 读取数据
使用 HttpClient 爬行分析 HTML 读取 WebLog。

* 如何向目标 BSP 写入数据
对于没有提供专用接口的，用模拟 Blogger 使用浏览器来发表 WebLog 的方式，即使用 HTTP Post 协议发送请
求来完成。该方法的缺点是不能保持源 WebLog 的发布时间，因为大部分BSP都不提供指定（修改） WebLog
的发表时间。

* CSDN Blog 定制的 BlogWriter
由 CSDN Blog 提供一个写入接口（比如 Web Service 接口，或者直接写入数据库的方式）来完成写入。

* 名词
WebLog：日志，一篇文章，包含评论，附件（如多媒体）等信息。可以缩写成Blog。
Blog Remover：Blog 搬家工具（搬家程序）。
BSP： Blog Service Provider，Blog服务提供商。
BlogReader：用来从源BSP读取WebLog的读取器。
BlogWriter：用来向目标BSP写入WebLog的写入器。
HttpClient：Apache 提供的一个 HTTP 客户端开发包（类似一个浏览器的实现）。
